# 🧠 0. 핵심 결론 (먼저)

👉 CorpBrain이 이길 수 있는 포지션은 명확하다:

### 🔥 **“Data + Knowledge + Execution을 SME에 맞게 압축한 통합 OS”**

* Unstructured: 데이터만
* Upstage: 이해만
* Allganize: 검색만
* GridOne: 실행만
* LlamaIndex: 개발자용

👉 **CorpBrain = “비개발자 SME용 올인원 실행형 AI”**

---

# 🧩 1. 기존 플레이어 대비 구조적 빈틈

## ❌ 현재 시장의 공통 문제

| 영역         | 문제                 |
| ---------- | ------------------ |
| Data       | 전처리만 해주고 끝         |
| RAG        | 검색은 되는데 실행 안됨      |
| Automation | 데이터 이해 부족          |
| Infra      | 비용/보안 trade-off 존재 |

👉 핵심 공백:

> **“데이터 정리 → 이해 → 활용 → 실행”이 하나로 안 이어짐**

# 🚀 2. CorpBrain 차별화 가치사슬 (핵심 설계)

## 🔗 통합 Value Chain

```id="corpbrain-flow"
[Data Chaos]
   ↓
[① Data Cleaning & Structuring]
   ↓

   ↓
[③ Decision Engine]
   ↓

   ↓
[Business Outcome]
```

👉 핵심:
**“검색이 아니라 실행까지 이어지는 구조”**

# 3️⃣ 가치사슬 주요활동 (차별화 중심 재설계)

---

## ① Inbound: 데이터 클렌징 & 구조 재설계 (🔥 핵심 차별화)

👉 경쟁사:

* Unstructured → parsing만
* Upstage → 인식만

### 🏗️ 설계

* 기능:

  * 중복 문서 제거 (semantic deduplication)
  * 폴더 구조 자동 재구성
  * 문서 중요도 scoring
* 결과:
  👉 “정리된 데이터 자산”

### 🚀 핵심 혁신

* **“파일 시스템 자체를 재설계”**
* Knowledge Graph 자동 생성

👉 💥 승부 포인트
➡️ **단순 ingestion이 아니라 ‘데이터 컨설팅 자동화’**

---

## ② Operations: Hybrid Knowledge Engine

👉 경쟁사:

* Allganize → RAG만
* LlamaIndex → 개발자용

### 🏗️ 설계

* 구조:

  * page-level chunking
  * knowledge graph 결합
  * hybrid retrieval (vector + symbolic)

### 🚀 혁신

* context-aware retrieval (업무 맥락 반영)
* multi-document reasoning

👉 💥 승부 포인트
➡️ **“검색 정확도”가 아니라 “업무 이해도”**

---

## ③ Decision Layer (🔥 신규 레이어)

👉 기존 기업에 없는 영역

### 🏗️ 설계

* 기능:

  * 질문 → 단순 답변이 아니라
  * **“추천 + 판단 + next action” 제공**

예:

* “이 계약 승인해도 되나?”
  →
* 리스크 분석
* 관련 규정 근거
* 승인 여부 추천

### 🚀 혁신

* rule + LLM hybrid decision engine
* confidence scoring

👉 💥 승부 포인트
➡️ **“검색툴 → 의사결정 엔진”**

---

## ④ Execution Layer (GridOne과 차별화)

👉 경쟁사:

* GridOne → RPA 중심

### 🏗️ 설계

* 기능:

  * 이메일 발송
  * 문서 생성
  * ERP 입력
  * 승인 workflow 실행

### 🚀 혁신

* RPA 없이 API 기반 lightweight automation
* agent 기반 실행

👉 💥 승부 포인트
➡️ **“RPA 없이도 실행되는 AI”**

---

## ⑤ Delivery: SME 최적 UX

### 🏗️ 설계

* 인터페이스:

  * Chat + Dashboard
  * 업무별 template

### 🚀 혁신

* “업무 중심 UI”

  * ex) 계약관리, 인수인계, 고객응대

👉 💥 승부 포인트
➡️ **툴이 아니라 ‘업무 환경’**
