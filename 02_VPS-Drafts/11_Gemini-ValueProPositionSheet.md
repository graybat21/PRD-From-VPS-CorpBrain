# 11_Value_Proposition_Sheet (가치 제안 통합 명세서)

**작성 목적**: 예비창업자가 복잡한 시장 분석 및 고객 인터뷰 결과(TAM-SAM-SOM, 페르소나, 여정지도, AOS-DOS, JTBD 등)를 관통하는 **'어떤 고객에게, 어떤 차별적 가치를 제안할 것인가?'**를 직관적인 하나의 프레임워크로 완성하여 투융자 및 제품 개발의 나침반으로 삼기 위함.
**대상 독자**: 첫 신규 사업(B2B 비정형 데이터 처리 솔루션)을 기획하는 예비창업자 및 프로덕트 리더

## 1. Value Proposition Sheet (통합 버전)
> **💡 다온 코멘트**: 회비서님, 초창기 창업 리소스는 매우 한정되어 있습니다. 거창한 만능 비전보다는 '당장 고객이 지갑을 열게 만들 뾰족한 가치'에 집중해야 합니다. 본 시트는 환각 공포에 시달리는 **[부티크 로펌]**과 지식 유실에 직면한 **[기술 스타트업]** 두 가지 핵심 거점 시장(Target Core)별로 완벽히 분리 및 맞춤 정리되었습니다.

### 🎯 핵심 타겟 1: 이지언 (부티크 특화 로펌 대표/파트너)
*데이터 유출과 문서 오판이 기업 생존을 좌우하는 전문 고수익 직군*

| 항목 | 상세 내용 |
| :--- | :--- |
| **페르소나 및 핵심 문제 (Pain, Needs)** | **요약**: 챗봇(범용 모델)이 특수 양식(표, 도장)을 박살내어 치명적 오판 리스크 발생. 극단적인 로컬 망분리 강박 기저.<br>**Needs**: 한 치의 오차도 없는 원본 구조(표) 유지, 망분리 환경 내 철저한 보안 통제. 변호사의 수검수 반복 업무 철폐. |
| **고객 현 상황에 따른 목표 (Goal, Job)** | **JTBD**: "(상황) 복잡한 도표나 특수 양식이 포함된 서류로 긴급하고 중대한 법률적 의사결정을 내려야 할 때, (목표) **단 1%의 정밀도 손실 없이 이를 완벽한 정형 데이터로 빠르게 구조화**하고자 함." |
| **고객이 원하는 Outcome** | 1. 변호사의 수작업 문서 검수 시간 대폭 단축 (일평균 4시간 → 30분 이내)<br>2. 표/특수 양식 내 핵심 텍스트 환각율 및 오표기율 0건 통제<br>3. 사내망(망분리) 보장으로 정보 유출 리스크 일체 차단 (100% 로컬) |
| **우리의 핵심 가치 제안 (Value Proposition)** | **"특수 구조 무결점 파싱과 오류 검증(Confidence) 시각화 엔진을 갖춘, 법무/회계 전용 로컬망 데이터 자산화 솔루션"** |
| **기존 대안 (Competitor / Substitute)** | - **경쟁 대안**: 업스테이지, 올거나이즈 등 대형사 타겟의 고가 구축형 버티컬 솔루션.<br>- **고객의 워크어라운드**: 해답 부재. 범용 OCR 대조 후 주니어 변호사를 동원해 일일이 엑셀에 타이핑/육안 전수 검사. |
| **제공하는 차별적 가치** | - **정밀도 역량 입증**: 고비용 대기업 솔루션에 상응하는 가장 강력한 'Table/Form 추출' 로직 엔진.<br>- **심리적 안도감 (UX)**: AI 스스로 확신도가 떨어지는 데이터만 '붉은색' 등으로 띄워, 인간 변호사의 전수 검사 강박 해소 및 효율 극대화. |
| **Proof (근거 / 검증 데이터)** | - **1순위 우선 타겟**: AOS(니즈) 4.0 / DOS(시장성) 3.60 기록.<br>- 지불 용의 최상위 입증 ("숫자 밀리면 의견서 전체가 휴지통, 맞기만 하면 즉시 결제" 인터뷰 발췌) |

| 항목 | 상세 내용 |
| :--- | :--- |

| **고객 현 상황에 따른 목표 (Goal, Job)** | **JTBD**: "(상황) 인력 교체기나 폭발적 성장에 따른 레거시 지식 단절 위기 시, (목표) **직원들에게 신규 툴 학습이라는 마찰을 가하지 않고 흩어진 구버전 정보 중 최신 유효 문서만을 큐레이션한 RAG 체계를 구축**하고자 함." |

| **기존 대안 (Competitor / Substitute)** | - **경쟁 대안**: Unstructured.io, LlamaIndex 등 개발자 친화적 인프라 솔루션 (본업 외 러닝커브와 커스텀 노력 필요).<br>- **고객의 워크어라운드**: 개발팀장 개인이 수작업으로 링크를 긁어내 스프레드시트나 노션에 노가다로 정리. |

## 2. JobMVP Feature Map (기능 우선순위 정리)

아래는 상기 Value Proposition을 기술적으로 실현하기 위해 당장 첫 번째 이터레이션(Iteraton 1)에서 개발해야 할 제품(MVP)의 백로그 우선순위입니다. 본 기능들은 철저히 **시장 매력도와 지불 용의(DOS)가 보증된 순서**로 나열되었습니다.

| MVP 우선순위 | 핵심 기능 스펙 (JobMVP Feature) | 대상 고객 페르소나 | 기대 효과 및 구현 맥락 제언 |
| :---: | :--- | :--- | :--- |
| **P 1** | **무결점 Table & Form 파서 (Parser) 코어 로직** | 이지언 (부티크 로펌 / 회계) | 텍스트만 긁어내는 수준을 넘어 원본 문서의 틀과 셀 매트릭스를 100% 보존. 대규모 범용 오픈소스(GPT 포함)와 당사 솔루션을 차별화 짓는 최초의 기술적 해자입니다. |
| **P 2** | **에러 검출 (Confidence) 하이라이터 UI** | 이지언 (부티크 로펌 / 회계) | AI가 추론 신뢰도가 떨어지는 구역만 컬러링하여 대시보드에 표기. 인간(전문가)이 병목 구역만 전수 검수하게 유도함으로써 AI 맹신 공포를 지우는 가장 즉각적인 세일즈 기믹(결제 유도 무기)입니다. |
| **P 3** | **API One-Click 백그라운드 커넥터** | 김동현 (성장기 스타트업 CTO) | Slack, Github 등 기존 워크스페이스의 데이터를 백그라운드에서 증분(수정본)만 추적하여 최신화. 임직원의 '행동 변화(이주)' 마찰을 완전히 제거하는 핵심 브릿지 장치입니다. |
| **P 4** | **PII (개인정보) 오토 마스킹 컴플라이언스 엔진** | 공통 (모든 B2B 조직 도입 기저) | 고객 온프레미스/VPC 배포 시 사내/고객 관련 민감 정보를 자동 블라인드 처리. 기능적 차별점을 떠나, B2B 엔터프라이즈의 기초 Compliance 장벽을 뚫기 위해 필수불가결합니다. |
| **Drop** | *(당장 개발하지 마십시오)* 음성/사진 기반 무마찰 제로 입력기 | 오석동 (제조업 공장장) | DOS 지표 상 시장 확산성이 최하(0.96)로 확인되었습니다. 타겟 페르소나의 결핍은 강렬하나 해당 생태계의 채택 속도가 느려 런웨이에 치명상을 줍니다. Q2 니치 트랩으로 간주해 보류합니다. |

## 3. 예비창업자 회비서님을 위한 다온의 전략 어드바이스

성공적인 시장 안착 및 피벗 방어선을 구축하기 위해 덧붙이는 제언입니다.

1. **'가짜 스케일업'인 커스텀(SI) 함정을 냉정하게 거절하십시오.**
   - 은행 특화 거대 인프라 수요자(최재벌 페르소나)가 찾아와 수십억 원짜리 전면 맞춤형 시스템 구축을 요구할지라도 응해서는 안 됩니다. 한정된 초기 리소스를 본질적 OS 엔진 고도화가 아닌 특정 1개사를 위한 하청 용역에 태우는 순간 '플랫폼 스타트업'은 사망합니다.

2. **결제 용의가 뾰족한 로펌/회계 타겟의 성과를 레버리지 하십시오.**
   - 비록 규모는 작아 보여도, 로펌 시장은 돈을 낼 의지와 동기가 가장 확고한 'Q1(1사분면) 스윗스팟'입니다. 이들의 혹독한 오차 기준을 넘긴 파서(Parser)를 확보하면, 이후 스타트업 CTO를 거쳐 전 범위의 일반 기업으로 자연스레 기능성을 과시(증명) 받으며 확장할 수 있습니다.

3. **개발팀을 자리에 앉히기 전에 랜딩 화면부터 들고나가십시오.**
   - 본 문서로 가정이 성립되었습니다. 즉시 이를 바탕으로 한 '가짜 UI 화면이 들어간 랜딩페이지'나 '미팅용 세일즈 덱(소개서)'를 작성해야 합니다. 이후 실제 특화 로펌 대표나 시리즈 B CTO들의 연락처를 찾아 당장 콜드콜 미팅(Pre-totyping)을 꽂아 실제 구매 전환율을 확인하는 것이 코딩보다 더욱 시급한 제1순위 과제입니다.
